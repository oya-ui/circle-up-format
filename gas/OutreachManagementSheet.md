# 声掛け管理シート（OutreachManagementSheet.gs）の説明

`OutreachManagementSheet.gs` は **プログラムの中身** であり、Google のサーバー上で動かすまでスプレッドシートはできません。ここでは **何が起きるか** と **あなたがやる操作** だけを書きます。

---

## これは何をするもの？

**新しい Google スプレッドシートを1つ自動で作る** スクリプトです。

| 項目 | 内容 |
|------|------|
| ファイル名（Googleドライブ上） | `サークルアップ連携PJ｜声掛け管理` |
| 中のシート名 | `声掛け管理` |
| 中身 | 1行目に列の見出し、2行目以降は空（あなたが団体を1行ずつ入れる） |

見出し行は **太字・薄い青背景**、**1行目固定**（スクロールしても見出しが残る）になっています。

---

## あなたがやる手順（コピペでOK）

1. ブラウザで [Google Apps Script](https://script.google.com/) を開く。
2. 「新しいプロジェクト」を作る（名前はなんでもよい）。
3. 左の `コード.gs` を開き、**中身をすべて消して**、ローカルの `OutreachManagementSheet.gs` の **全文をコピーして貼り付け**。
4. **保存**（フロッピーアイコンまたは Ctrl+S / Cmd+S）。
5. 上の関数プルダウンで **`createOutreachManagementSheet`** を選ぶ。
6. **実行**（▶）を押す。
7. 初回だけ「権限を確認」→ Google アカウントを選び、**スプレッドシートの作成を許可**する。
8. メニュー **表示 → ログ**（または Ctrl+Enter 実行後のログ）で **`声掛け管理スプレッドシート: https://...`** と出るので、そのURLを開く。  
   そのURLを `運用_リンク一覧.md` などにメモしておくとよいです。

**注意:** この「実行」を押すたびに **別の新しいスプレッドシート** が増えます。本番用は **1回作ったら二度三度と実行しない** のが安全です。

## 列の意味（A列〜K列）

コード上は「1行目に次の文字を並べる」と決め打ちしています。

| 列 | 見出し | 何を書くか | セルのルール（ざっくり） |
|----|--------|------------|---------------------------|
| A | 団体ID | 例: C001。運用で連番でもよい | 特になし（自由） |
| B | 正式名称 | サークルの正式名 | 自由 |
| C | 大学 | 拠点の大学名など | 自由 |
| D | エリア | 仙台 / 東京 / 札幌 / その他 | **この4つ以外は入れられない**（ドロップダウン） |
| E | ステージ | 下の一覧から選ぶのがおすすめ | ドロップダウンあり。**リストにない言葉も手入力できる**（運用で増やしたいとき用） |
| F | 最終接触日 | いつ最後に動いたか | **日付として入れる想定**。空欄や別の書き方も許す設定 |
| G | 次アクション日 | 次いつやるか | 上に同じ |
| H | 担当 | 名前 | 自由 |
| I | チャネル | どう繋がったか | ドロップダウン（メンター紹介 等）。**別の言葉も入力可** |
| J | 反応 | 相手の温度感 | ドロップダウン（前向き 等）。**別の言葉も入力可** |
| K | 次の一手メモ | 次にやることのメモ | 自由（列幅広め） |

### ステージ（E列）の初期リスト

スクリプトが最初から用意している候補です。

- 未接触  
- 接触済  
- 説明予定  
- 説明済  
- フォーム送付済  
- 保留  
- 辞退

## 「コードがやっていること」を一言ずつ（読み飛ばしOK）

| コードのイメージ | 人間語 |
|------------------|--------|
| `SpreadsheetApp.create` | 新しいブックを作る |
| `setName('声掛け管理')` | 最初のシートの名前を変える |
| `headers` の配列 | 1行目の見出しの並び順 |
| `setFrozenRows(1)` | 1行目を固定 |
| `setColumnWidth` | 列の幅をだいたい見やすくする |
| `newDataValidation() ...` | ドロップダウンや「日付推奨」をセルに付ける |
| `setAllowInvalid(false)` | リスト外は **拒否**（D列エリアだけ） |
| `setAllowInvalid(true)` | リスト外も **許す**（ステージ・日付・チャネル・反応） |
| `Logger.log` | 実行後、ログにURLを出す |

---

## うまくいかないとき

- **実行してもログにURLが出ない**  
  → 「表示 → ログ」を開き直す。エラーが出ていれば赤文字を読む（権限未承認が多い）。

- **日付列に変なエラーが出る**  
  → F・G列は「日付型」を推奨する設定です。「3/30」のような入力や空欄は許容設定ですが、環境によっては警告が出ることがあります。そのときはセルの書式を「日付」にするか、テキストで書きたい列に変えたい場合は GAS の該当行を外す（要相談）で対応できます。

- **列を増やしたい・エリア名を変えたい**  
  → `OutreachManagementSheet.gs` の `headers` 配列と、`requireValueInList` の中身を編集してから、**新しいシート用に** もう一度実行するか、既存シートを手で列追加してください。
