サークルアップ × Geek 連携PJ｜長期インターン向けオペレーション説明

この資料の目的
面談・打ち合わせまでの導線をつないでいただく長期インターンの方が、プロジェクトの全体像と、自分がやるべきことを迷わず実行できるようにまとめたものです。前提知識は不要です。

1. このPJの目的と背景

1.1 サークルアップ（Circle Up）とは

  • 何のサービスか
    企業の案件（イベント参加で報酬がもらえる、など）と大学のサークルをつなぐマッチング型のプラットフォームです。
    公式: https://www.circle-app.jp/
  • 誰のためのサービスか（ざっくり）
    • 企業側: 若年層（主に大学1〜3年程度）へのリーチ、就活イベントへの参加依頼、美容・着物など季節ニーズへの露出、など。
    • サークル側: 活動費や新歓の資金が欲しい、協賛を探したい、など。
  • 収益のイメージ
    企業が案件に払う金額に対し、運営が手数料をいただくビジネスモデルです（詳細は社内説明に準拠）。

1.2 Geek・エンカレッジとの関係

  • Geek（ジーク）
    プログラミング学習・長期インターンなどを扱うブランド／コミュニティです。このPJでは、サークル幹部とGeekのメンター／長期インターンとの接点を増やすことが目的の一つです。
  • エンカレッジ（en-courage）
    就活支援などを行うサービス。メンター経由でサークル幹部にアプローチする経路として使います。

1.3 なぜこのPJをやっているか（背景）

  • サークルアップ事業を本格的に伸ばすフェーズにあり、実際にサークルに使ってもらうことが重要になっている。
  • 一方で、「サークルアップを使いたい」というニーズは団体・タイミングによって差が大きい（特にお金の管理に関わる役職に刺さりやすい、などの学びがある）。
  • そこで 新歓の時期の悩み（人数・認知・資金） に寄り添い、新歓用のWeb（紹介ページ）の代行をフックにして接点を作り、その流れの中でサークルアップの利用とGeekの訴求につなげる、という設計です。

1.4 このPJで目指していること（一文）

サークル幹部との信頼できる接点をつくり、新歓に役立つ納品（Web）を通じてサークルアップ導入とGeekの認知・採用導線に結びつけること。

2. インパクト（何がうれしいか）

2.1 サークル（幹部）にとって

  • 新歓で使えるサークル紹介のWebが整い、外部に見せる窓口ができる。
  • 活動費・新歓費の足しに、サークルアップ経由の案件を検討できるようになる可能性がある。

2.2 サークルアップ事業にとって

  • 実利用の団体が増え、掲載・案件の両面でデータと認知が積み上がる。

2.3 Geekにとって

  • サークル幹部（リーダー経験のある学生）との自然な接点ができる。
  • 長期インターンやメンター紹介など、採用・コミュニティ拡張に使える。

2.4 あなた（長期インターン）にとって

  • 営業・プロジェクト推進・ステークホルダー調整の実践になる。
  • 成果に応じたインセンティブ設計がある場合は、別紙の契約・説明に従う（※金額・条件は担当から必ず確認すること）。

3. リスク・注意点（事前に知っておくこと）

3.1 ニーズが必ずあるわけではない

「協賛を探しているフェーズ」ではない団体や、予算・会計に関わる人に届いていない場合、サークルアップの話は響きにくいことがあります。断られて当然と考え、無理に押し売りしないこと。関係が壊れないよう、丁寧なコミュニケーションを優先してください。

3.2 「HP代行」単体だけが目的に見えると危険

ペライチ等ですでに満足している団体もあります。訴求の軸は 「新歓の入口・認知のためのページ」 や 「新歓・運営の困りごとに寄り添う」 であることが重要です（過去の合意事項に準拠）。

3.3 説明不足・フォロー不足

インフル・多忙などで計画の全体像が伝わらないまま止まると、後から信頼を損ねます。再日程・フォローを自分のタスクとして管理してください。

3.4 技術・ツールの制約

Web制作には Cursor・GitHub・Netlify等 を使う想定があり、学生プラン申請に時間がかかる場合があります。納品時期は現実的なスケジュールで幹部と合意してください。

3.5 個人情報・連絡先

取得した電話番号・LINE・メール等は、目的外利用をせず、社内ルール（個人情報・営業ツールの取り扱い）に従ってください。

4. やってほしいこと（役割の全体像）

あなたの役割はざっくり 次の流れを途切れさせないこと です。

  1. サークル幹部に打診する（新歓用Webの代行・サークルアップの導線の説明）。
  2. 依頼情報を集める（フォーム提出の促し）。
  3. LINEグループ（LG）等の場をつくり、関係者をつなぐ。
  4. 納品の打ち合わせの日程を決め、幹部が必ず参加する状態にする。
  5. 打ち合わせの場で、サークルアップ担当・Geek側の訴求ができるよう設計に関与する（台本・タイミングは後述）。

※ 実際のWebの制作作業を誰が行うか（メンター／別担当／あなた）は、その時点の運用ルールに従ってください。不明なときは必ず担当に確認してください。

6. 幹部への打診コミュニケーション（例と心構え）

6.1 心構え

  • 押し売りではなく「新歓の役に立てれば」というスタンス。
  • 最初から長い説明をせず、興味があれば詳しく話す二段構えにすると返信が返りやすいです。
  • 幹部は多忙なので、一文目でメリットが伝わるようにする。

6.2 伝えたい核（必ず伝わるように）

  1. 新歓で使える紹介ページを、こちらで代行で作る（または一緒に作る）提案があること。
  2. 情報をフォームで送ってもらえれば、打ち合わせで方向性をすり合わせること。
  3. 打ち合わせではサークルの代表・新歓関係の役職の方に参加してもらうこと（2名以上が望ましい、という条件がある場合はその旨を明確に）。
  4. あわせて、活動費を補う仕組みとしてサークルアップの紹介があること（詳細は専門担当が説明する）。

7. 納品の場（MTG）の設定

7.1 なぜ「場」を必ず設定するか

  • 画面で確認しながら認識のズレを防ぐため。
  • サークルアップ・Geekの説明を、幹部がいる状態で行うため（録画ポリシーは事前に合意）。

7.2 参加者（理想）

  • サークル側: 代表クラス＋新歓・広報まわりなど、最低2名（プロジェクトでそう決めている場合）。
  • あなた（インターン）: 進行補助、フォーム内容の確認。
  • Geek担当・メンター（制作担当がいる場合）。
  • サークルアップ事業担当（導入説明をする人）。
  • 社内オーナー（必要に応じて）。

7.3 幹部に来てもらうコツ

  • 日程調整は 候補を3つ出す。
  • 「決定権のある人がいないと次に進めない」理由を一言添える。
  • 前日に リマインド（LINE／カレンダー招待）。

8. MTGの流れと「訴求」の設計（サークルアップ・Geek）

8.1 おすすめの時間配分（例：60分）

時間内容
0〜5分挨拶、参加者確認、今日のゴール（納品確認＋サービス紹介）
5〜35分納品物の確認（トップ、写真、文章、連絡先、スマホ表示）
35〜50分サークルアップの説明（事業担当）— 幹部の質疑
50〜60分Geek長期インターンの紹介（自然な流れで。興味ある人向け）

※ 納品が長引く日は、サークルアップを先に短くしてもよいなど、担当と事前に合意してください。

8.2 訴求の順番の理由

  1. まず納品で信頼と文脈を作る（「約束したことをやった」）。
  2. その後で活動費の話（サークルアップ）が入りやすい。
  3. キャリア・スキル（Geek）は、幹部個人の関心が出たタイミングで深掘りしやすい。

8.3 Geekを話すときのポイント

  • 押し込みではなく「知っておいてほしい選択肢」として。
  • サークル運営で培った経験と、インターンで得られる経験がつながる話がしやすい。
  • 詳細は Geek側の公式説明・スライドがあればそれに任せ、あなたはファシリテーションに徹する。

8.4 サークルアップを話すときのポイント

  • 事業担当が主役。あなたは幹部の不安（手間・料金・何をするか）を拾って担当に渡す。
  • 導入までのステップ（アカウント、掲載、案件の探し方）を最後に整理しておくと幹部が動きやすい。

9. 納品に関するオペレーション（実務チェックリスト）

9.1 納品前

依頼フォーム（回答用）
https://docs.google.com/forms/d/e/1FAIpQLSeFVBEoz5WxbuBQQ8pHczL3kePpmTCcD34eieh0Tgg8ynYjgQ/viewform（一覧: 運用_リンク一覧.md）

  • フォームの必須項目が揃っている（サークル名、紹介、活動内容、人数、代表連絡先など）
  • 写真はフォームでは受け付けずLINEアルバム等の前提のため、送付済み／送付予定のすり合わせができている
  • 画像の画質・横長縦長の指定があれば満たしている
  • 公開URLの掲載先（Netlify等）がわかる
  • 修正の一回目締切・連絡手段（LINE／メール）が決まっている

9.2 納品MTG中

  • スマホ表示を必ず見せる
  • 誤字・サークル名・連絡先の最終確認
  • 「公開してよいか」の言葉でのOKをもらう

9.3 納品後

  • 公開URLを幹部に送付
  • 軽い修正の期限を伝える
  • サークルアップの次のアクション（誰がアカウントを作るか、いつまでか）をメモ
  • Geek興味ありの学生がいれば担当へ引き継ぎ（連絡先・希望の確認）

9.4 トラブル時

クレーム・法的表現・写真の権利などは即担当にエスカレーション。一人で判断しない。

10. 困ったとき・質問するとき

  • 契約・報酬・インセンティブの数字 → 必ず事業担当／契約の窓口に確認。
  • サークルアップの仕様・案件内容 → サークルアップ事業担当。
  • Geekの採用・インターン詳細 → Geek／採用担当。
  • 日程・LGの作り方・誰を入れるか → このPJのオーナー（例：大谷・楠・教室長）。

11. 一文まとめ

幹部に誠実に打診し、フォームとMTGを確実に通し、納品の場でサークルアップとGeekの話がちゃんと届くように進行する——それが、このPJでのあなたのオペレーションの中心です。
