# サークルアップ連携PJ｜週次数値更新・声掛け分析

## 前提（このリポジトリ内の正）

- **G1・声掛けの実体**: 共有スプレッドシート（列定義は `gas/OutreachManagementSheet.gs`）。**閲覧可能な共有**のもと、CSV は `reference.md` の export URL から取得する。人間用URL・export URL は `運用_リンク一覧.md` にも記載。
- **フォーム回答の実体**: Googleフォームに紐づく**回答スプレッドシート**（列＝設問、行＝1回答）。フォームの回答用URLは `運用_リンク一覧.md` を参照。
- **資料用の見た目**: 同フォルダの `数値進捗_サマリー.html`（G1/G2/G3・エリア別の表）。数値は主に**手編集**で埋める。ヘッダの「更新日」はブラウザ表示時に当日日付へ書き換わる（`script` 内）。会議基準日を固定したい場合は、その日付を明示的にHTMLに書くか、開く日と会議日がずれる点に注意。

## 数値分析・集計を頼まれたとき

1. [reference.md](reference.md) の **CSVエクスポートURL** で最新データを取得する（ネットワークツール可）。
2. CSV をパースし、**ヘッダ行と列名**に従って件数・内訳（エリア、ステージ、担当など）を集計する。
3. **有効行のルール（固定）**: **B列「正式名称」が空でない行だけ**を数える（空白トリム）。団体IDだけの行は含めない。集計結果に「有効行の定義」を一言添える。
4. 結果を会話に表や箇条書きで返す。HTML 更新も依頼されていれば `数値進捗_サマリー.html` を整合させる。
5. export が失敗したら `reference.md` の「取得できないとき」に従い、ユーザーに共有設定または範囲コピペを依頼する。

## 作業チェックリスト

ユーザーにスプレッドシートを開けない場合は、**集計に必要な範囲をコピーした表**または**画面キャプチャ**の貼り付けを依頼する。

- [ ] **声掛け管理**: `reference.md` の CSV export で取得（失敗時はコピペ依頼）
- [ ] **フォーム回答シートを開く**（G2 等。フォーム編集 → 回答タブ → スプレッドシート、または既知のURL）
- [ ] **週次で見る指標を確定**（下記「フォームから取れる指標」および `reference.md` の対応表）
- [ ] **集計**（件数・ユニーク団体数・ステータス別内訳など。定義は `reference.md` に合わせる）
- [ ] **`数値進捗_サマリー.html` を更新**（G1行のエリア別実績、エリア別表の `__ / 10`、G2/G3の実績セル、必要なら一言サマリー）
- [ ] **運用メタの更新**（数値の出所やフォームURLを変えた場合は `運用_リンク一覧.md` の最終更新日・URLを確認）

## フォームから取れる指標（例）

実際の列名はスプレッドシートで確認する。HP依頼フォーム（`gas/CircleHpRequestForm.gs` で定義）から典型的に取れるのは次のようなもの。

| 指標の例 | 取り方の例 |
|----------|------------|
| 依頼件数（累計・週次） | タイムスタンプ列で期間フィルタして行数 |
| ユニーク団体数 | 「サークル名（正式名称）」列の重複除外 |
| 大学・エリアの分布 | 「拠点となる大学名」等から手動でエリアに寄せる、またはピボット |
| パイプライン（G2） | 依頼〜納品の**社内トラッキング列**がある場合はそれを使用。無い場合は「フォーム送信数＝候補母数」と定義しメモに明記 |

**G1（声掛け）**はフォーム外の活動量であることが多い。フォームに該当設問が無い場合は、スプレッドシートではなく**別管理表・口頭確定の数**をHTMLに反映し、メモ欄に出所を書く。

## HTML を触るときの注意

- 表の構造（`th`/`td`・`class="num"`）を壊さない。
- プレースホルダ `__` を数値または短文に置き換える。
- 文言修正が必要なら、数値だけでなく「状況サマリー」「次の一手」も会議向けに整える。

## 追加のドメイン定義

列名とKPIの対応・週の切り方（例: 月曜締め前週月〜日）を固定したくなったら、[reference.md](reference.md) に追記する。
